# universal_agents

Самодостаточный фреймворк LLM-агентов с вызовом инструментов, написанный на
Python. Запускает одного основного `LLMAgent` (и при необходимости
`SubAgent`) поверх OpenAI-совместимого API, позволяет модели вызывать
инструменты и включает ряд механизмов самоуправления: обнаружение
зацикливаний, автосжатие вывода инструментов, бюджет контекста, восстановление
истории, self-consistency и изолированную песочницу.

Вся система рассчитана на локальный запуск — по умолчанию она общается с
сервером [LM Studio](https://lmstudio.ai/) (`http://localhost:1234/v1`).

## Возможности

- **Динамическое управление инструментами** — LLM может включать/отключать
  инструменты через `load_tools` и `disable_tool`, самостоятельно управляя
  своим инструментарием в процессе работы.
- **Плагинные инструменты** — достаточно положить `.py`-файл в `tools/`,
  использующий декоратор `@tool`, и он подгрузится при запуске автоматически.
- **Субагенты** — делегируют самодостаточную подзадачу изолированному агенту
  со своей историей и бюджетом токенов, опционально ограниченному безопасными
  (только для чтения) инструментами.
- **Предотвращение зацикливаний** — обнаруживает и блокирует повторные вызовы
  одного инструмента с теми же аргументами в рамках хода (как проактивно до
  выполнения, так и реактивно во время выполнения).
- **Автосжатие** — длинные выводы инструментов автоматически суммируются по
  частям субагентами до сохранения в историю, освобождая токены контекста.
- **Бюджет контекста** — `TokenUsageTracker` оценивает оставшиеся токены и
  внедряет заголовки с расходом токенов в промпт; инструменты позволяют
  агенту суммировать/удалять/редактировать историю, чтобы уложиться в бюджет.
- **Восстановление истории** — нормализует последовательность ролей сообщений
  и удаляет неудачные вызовы инструментов после ошибок, сохраняя валидность
  тела запроса к API.
- **Режим self-consistency** — генерирует несколько вариантов рассуждений и
  синтезирует итоговый ответ (переключается через `/consistent`).
- **Изолированное выполнение** — `bash`/`python` выполняются внутри
  изолированного Docker-контейнера с ограничением ресурсов и без сети.
- **Интерактивный CLI** — насыщенный консольный интерфейс с подтверждением
  операций, меняющих состояние, и набором slash-команд.

## Архитектура

```
universal_agents/
├── main.py              # Точка входа: сборка агента, загрузка плагинов, запуск CLI
├── agent.py             # LLMAgent: основной цикл диалога и выполнение инструментов
├── sub_agent.py         # SubAgent: изолированный, опционально безопасный делегат
├── config.py            # URL API, имя модели, пороги
├── llm_client.py        # Обёртка над OpenAI + TokenUsageTracker + LoopDetector
├── tool.py              # Декоратор @tool и генерация JSON-схемы
├── tool_registry.py     # Загрузка функций с @tool из директории tools/
├── models.py            # Датаклассы сообщений (System/User/Assistant/ToolResult)
├── history.py           # ChatHistory: добавление/редактирование/удаление/суммаризация/сохранение/загрузка
├── context_builder.py   # Подготовка истории для API (заголовки, prefill)
├── history_repair.py    # Разрыв циклов и очистка неудачных вызовов
├── compressors.py       # Суммаризация на базе LLM и порционный анализ
├── ui.py                # Консольный рендерер и интерактивный CLI
└── tools/
    ├── builtin.py       # get_messages, edit_message, delete_messages,
    │                     #   summarize_messages, delegate_to_subagent
    ├── fs.py            # cwd, read, edit_file (работа с файловой системой)
    └── sandbox.py       # start_sandbox, run_bash, run_python, stop_sandbox
```

### Основной поток данных

1. `main.py` загружает внешние инструменты через
   `load_external_plugins("tools")` (`tool_registry.py`) и создаёт `LLMAgent`.
2. CLI (`ui.py`) считывает ввод пользователя и вызывает `agent.chat(...)`.
3. `agent.chat` в цикле: собирает сообщения для API (`context_builder.py`),
   вызывает модель (`llm_client.py`), обрабатывает ответ, выполняет вызовы
   инструментов (`agent._execute_tools`) и добавляет результаты в историю.
4. Выводы инструментов сжимаются на лету (`compressors.py`), когда превышают
   порог токенов относительно оставшегося бюджета.
5. После каждого хода история восстанавливается/нормализуется
   (`history_repair.py`, `history.py`).

## Требования

- Python 3.11+
- Python SDK `openai`
- Docker (нужен только для инструментов песочницы в `tools/sandbox.py`)
- Локальный OpenAI-совместимый сервер, например LM Studio, доступный по
  `http://localhost:1234/v1` (настраивается в `config.py`)

Установка зависимостей:

```bash
pip install openai
```

## Запуск

```bash
cd universal_agents
python main.py
```

Затем введите сообщение и нажмите Enter. Для выхода введите `exit` или `quit`.

### Команды CLI

| Команда         | Описание                                                   |
|-----------------|------------------------------------------------------------|
| `/regen [n]`    | Перегенерировать последний ответ `n` раз (по умолчанию 1) |
| `/prefill <txt>`| Заставить следующий ответ ассистента начаться с `<txt>`    |
| `/save [file]`  | Сохранить историю диалога в JSON-файл                     |
| `/load [file]`  | Загрузить историю диалога из JSON-файла                   |
| `/consistent`   | Переключить режим self-consistency (синтез по нескольким вариантам) |
| `/multiline`    | Переключить многострочный ввод (завершается пустой `/mm`) |

## Конфигурация

Отредактируйте `config.py`:

```python
class Config:
    API_URL = "http://localhost:1234/v1"  # OpenAI-совместимый эндпоинт
    MODEL_NAME = "local-model"             # Идентификатор модели, известный эндпоинту
    AFTER_SYSTEM_PROMPT = 1                # Индекс, с которого начинается диалог
    BOOST_TEMP = 0.875                      # Температура при восстановлении после циклов/пустых ответов
```

Параметр `max_context_tokens` агента (например, `50000` в `main.py`) задаёт
бюджет токенов, на основе которого работают автосжатие и внедряемые
заголовки с расходом токенов.

## Написание своего инструмента

Создайте `.py`-файл в `tools/` и отметьте функцию декоратором `@tool`.
Подсказки типов (`str`, `int`, `float`, `bool`, `list`, `dict`)
автоматически преобразуются в JSON Schema. Инструменты, меняющие состояние,
помечайте `requires_confirmation=True`, чтобы запрашивать подтверждение у
пользователя перед выполнением.

```python
from universal_agents.tool import tool

@tool(
    description="Поприветствовать человека по имени",
    name=("str", "Имя для приветствия"),
)
def greet(name: str) -> str:
    return f"Hello, {name}!"
```

- Функция, чей первый параметр назван `agent` (либо зарегистрированная как
  метод экземпляра), получает запущенный `LLMAgent` и может использовать его
  историю.
- Функция с первым параметром `agent` также может вызывать
  `delegate_to_subagent` для делегирования подзадач.
- Возвращайте `str`; он станет содержимым результата инструмента.

Инструменты загружаются автоматически при запуске — этап регистрации не нужен.

## Инструменты по умолчанию

**Встроенные (`tools/builtin.py`)**
- `get_messages` — краткий индексированный вид текущей истории
- `edit_message` *(подтверждение)* — отредактировать сообщение по id
- `delete_messages` *(подтверждение)* — удалить диапазон сообщений
- `summarize_messages` *(подтверждение)* — сжать диапазон в одно саммари-сообщение
- `delegate_to_subagent` — передать подзадачу изолированному безопасному субагенту
- `load_tools` — включить ранее отключённый инструмент по имени; без аргументов показывает список доступных для включения инструментов
- `disable_tool` — отключить активный инструмент по имени (нельзя отключить `get_messages`, `load_tools`, `disable_tool`)

**Файловая система (`tools/fs.py`)**
- `cwd` — получить/сменить рабочую директорию
- `read` — прочитать файл (или диапазон строк), либо структурное саммари большого файла
- `edit_file` *(подтверждение)* — точная замена подстроки / создание файла с выводом diff

**Песочница (`tools/sandbox.py`)** — требует Docker
- `start_sandbox` — запустить изолированный контейнер для репозитория
- `run_bash` — выполнить bash-команду в контейнере
- `run_python` — выполнить Python-код в контейнере
- `stop_sandbox` — остановить и удалить контейнер

## Замечания по безопасности

- Песочница запускается с `--network none`, `--memory 1g`, `--cpus 1`,
  сброшенными capabilities и `no-new-privileges`. Она предназначена для
  недоверенного сгенерированного моделью кода, но требует проверки перед
  реальным использованием.
- Инструменты с `requires_confirmation=True` приостанавливаются и запрашивают
  `y/n` перед выполнением. Субагентам можно задать `safe_only=True`, чтобы
  отбросить все такие инструменты.
- Фреймворк предполагает локальный однопользовательский запуск; он **не**
  адаптирован для многопользовательского или удалённого доступа.
